Understanding Union Types in TypeScript

Union types are a powerful way to express a value that can be one of several types. You define a union type using the pipe character (|) between types. This allows for flexibility in function parameters, variables, and return types while maintaining type safety.

Unions are especially useful in scenarios like handling API responses where data might vary in structure, or when dealing with multiple possible input formats. They work seamlessly with type narrowing techniques like type guards to ensure safe operations on the values. 

// Defining a union type
type ID = string | number;

// Function that accepts a union type
function printID(id: ID) {
    if (typeof id === "string") {
        console.log(id.toUpperCase());
    } else {
        console.log(id.toFixed(2));
    }
}

printID("abc123"); // Output: ABC123
printID(123.456);  // Output: 123.46

// Advanced usage with objects
type Success = { status: 'success'; data: any };
type Error = { status: 'error'; message: string };
type Response = Success | Error;

function handleResponse(res: Response) {
    if (res.status === 'success') {
        console.log(res.data);
    } else {
        console.error(res.message);
    }
}

Union types are particularly useful when working with values that might come in different forms, such as API responses where a field could be a string, number, or null. Combining unions with other TypeScript features like generics can create highly reusable and type-safe code.

Remember, when using union types, the compiler only allows access to members that are common to all types in the union unless you narrow the type first. This prevents runtime errors and encourages robust code design.

Type Guards for Runtime Safety

Type guards are expressions that perform runtime checks to guarantee the type of a value within a specific scope. They help you write type-safe code even when dealing with dynamic data. Type guards can be built-in (like typeof or instanceof) or user-defined.

They are crucial for handling union types effectively, allowing the TypeScript compiler to understand type narrowing in conditional branches. This leads to fewer type assertions and more reliable code. 

// Type guard using typeof
function isString(value: any): value is string {
    return typeof value === "string";
}

// Type guard using instance checks
function isDate(value: any): value is Date {
    return value instanceof Date;
}

// Custom type guard
interface Bird {
    fly(): void;
    layEggs(): void;
}

interface Fish {
    swim(): void;
    layEggs(): void;
}

function isBird(pet: Bird | Fish): pet is Bird {
    return (pet as Bird).fly !== undefined;
}

function move(pet: Bird | Fish) {
    if (isBird(pet)) {
        pet.fly();
    } else {
        pet.swim();
    }
}

// Advanced example with discriminated unions
type Circle = { kind: 'circle'; radius: number };
type Square = { kind: 'square'; side: number };
type Shape = Circle | Square;

function area(shape: Shape): number {
    switch (shape.kind) {
        case 'circle':
            return Math.PI * shape.radius ** 2;
        case 'square':
            return shape.side ** 2;
    }
}

Type guards are essential for working with union types and ensuring your code is both flexible and type-safe. They shine in scenarios like parsing JSON data or handling polymorphic objects.

Best practices include combining type guards with switch statements for discriminated unions, which provides exhaustive type checking and helps catch missing cases during compilation.

Optional Chaining and Nullish Coalescing

Optional chaining (?.) and nullish coalescing (??) are modern JavaScript features that TypeScript fully supports. They make working with nested properties and default values much cleaner. Optional chaining short-circuits if a value is null or undefined, preventing errors.

Nullish coalescing provides a way to fall back to a default only if the value is null or undefined, unlike the || operator which falls back for any falsy value. These features greatly improve code readability in real-world applications. 

interface User {
    name: string;
    address?: {
        street: string;
        city: string;
        zipcode?: string;
    };
}

const user: User = {
    name: "John Doe",
    // address is optional and might be undefined
};

// Without optional chaining
const city = user.address && user.address.city;

// With optional chaining (much cleaner!)
const cityWithChaining = user.address?.city;

// Nullish coalescing for default values
const zipcode = user.address?.zipcode ?? "00000";

console.log(city); // Output: undefined
console.log(cityWithChaining); // Output: undefined
console.log(zipcode); // Output: 00000

// Advanced usage with functions and arrays
const users: User[] = [];
const firstUserCity = users[0]?.address?.city ?? 'Unknown';

function getUser(id: number): User | undefined {
    // simulate fetch
    return id === 1 ? user : undefined;
}

const fetchedCity = getUser(2)?.address?.city;

These features help eliminate verbose null/undefined checks and make your code more readable and maintainable. They are particularly useful in React components or when working with optional API fields.

Note that optional chaining can be used with function calls (?.) and array access (?.[index]), making it versatile for various scenarios.

Generics in TypeScript: Reusable Type-Safe Code

Generics allow you to create components that work with any data type while providing type safety. They are like variables for types, enabling reusable functions, classes, and interfaces without losing type information.

Generics are fundamental in building libraries and frameworks, as seen in Array or Promise. They help avoid code duplication and ensure type consistency across your application. 

// Basic generic function
function identity<T>(arg: T): T {
    return arg;
}

const num = identity(42); // num is number
const str = identity("hello"); // str is string

// Generic class
class GenericNumber<T> {
    zeroValue: T;
    add: (x: T, y: T) => T;
}

// Constraints on generics
function loggingIdentity<T extends { length: number }>(arg: T): T {
    console.log(arg.length);
    return arg;
}

// Generic interfaces
interface KeyValuePair<K, V> {
    key: K;
    value: V;
}

const pair: KeyValuePair<string, number> = { key: "age", value: 30 };

// Advanced: Conditional types with generics
type ExtractType<T> = T extends (infer U)[] ? U : never;
type ArrayElement = ExtractType<string[]>; // string

Generics shine in collections, utility functions, and when building extensible components. You can add constraints using 'extends' to limit the types that can be used.

Advanced usage includes default type parameters, conditional types, and mapped types, which allow for powerful type transformations and utilities in large-scale applications.

Type Assertions: When You Know Better

Type assertions allow you to tell the TypeScript compiler to treat a value as a specific type. This is useful when you have more information about the type than TypeScript can infer, such as when working with DOM elements or third-party libraries.

Assertions come in two forms: 'as' keyword or angle-bracket syntax. They don't perform runtime checks but help during compilation. Use them sparingly, as overuse can lead to runtime errors if incorrect. 

// Basic assertion
const someValue: any = "this is a string";
const strLength: number = (someValue as string).length;

// Angle-bracket syntax (avoid in JSX)
const strLength2: number = <string>someValue.length;

// With DOM
const canvas = document.getElementById('main') as HTMLCanvasElement;
const ctx = canvas.getContext('2d'); // Now type-safe

// Const assertions (TypeScript 3.4+)
let x = { hello: 'world' } as const; // Readonly deep
// x.hello = 'foo'; // Error

// Advanced: Assertion functions
function assertIsString(val: any): asserts val is string {
    if (typeof val !== 'string') {
        throw new Error('Not a string');
    }
}

function process(val: any) {
    assertIsString(val);
    // val is now string
    console.log(val.toUpperCase());
}

Type assertions are handy for bridging JavaScript and TypeScript code, but prefer type guards where possible for better safety. In modern TypeScript, assertion functions can even narrow types like guards.

Remember, assertions are a way to override the compiler's inference - use them when integrating with untyped code or when the type system can't express the reality.

Type Predicates: Custom Type Narrowing

Type predicates are user-defined type guards that return a boolean and use the 'is' keyword in their return type. They allow TypeScript to narrow types in conditional blocks based on custom runtime checks.

They extend built-in type guards, enabling complex checks like property existence or value ranges. Predicates are powerful for domain-specific type narrowing in applications. 

// Basic type predicate
function isNumber(value: any): value is number {
    return typeof value === 'number';
}

function processValue(value: string | number) {
    if (isNumber(value)) {
        console.log(value.toFixed(2));
    } else {
        console.log(value.toUpperCase());
    }
}

// Complex predicate
interface Admin {
    role: 'admin';
    permissions: string[];
}

interface User {
    role: 'user';
}

type Person = Admin | User;

function isAdmin(person: Person): person is Admin {
    return person.role === 'admin';
}

function getPermissions(person: Person) {
    if (isAdmin(person)) {
        return person.permissions;
    }
    return [];
}

// Advanced: Predicate with generics
function hasProperty<T, K extends string>(obj: T, prop: K): obj is T & { [P in K]: unknown } {
    return prop in obj;
}

const obj: any = { name: 'John' };
if (hasProperty(obj, 'name')) {
    console.log(obj.name.toUpperCase());
}

Type predicates are ideal for creating reusable narrowing functions that can be composed. They improve code organization and type safety in large codebases.

Combine predicates with assertion functions for even more control over type flow. Always ensure the predicate's runtime check matches its type declaration to avoid false positives.
